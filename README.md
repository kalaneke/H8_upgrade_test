# 软件升级工具项目策划书

---

## 一、项目概述

### 1.1 项目背景
为实现产品软件的远程维护和版本迭代，开发一款基于Git版本管理的电脑端升级工具。用户通过电脑连接产品U盘，工具自动检测版本差异，从云端Git仓库下载最新升级包并完成U盘文件更新。

### 1.2 项目目标
- 实现基于Git的版本管理与同步机制
- 提供图形化界面，操作简便
- 支持IAP固件升级和资源文件增量更新
- 确保升级过程安全可靠

### 1.3 工作流程
```
┌─────────────┐     ┌─────────────┐     ┌─────────────────────┐
│ 连接产品U盘 │ ──→ │ 启动升级工具 │ ──→ │ 读取U盘版本信息      │
└─────────────┘     └─────────────┘     └──────────┬──────────┘
                                                   ↓
┌─────────────┐     ┌─────────────┐     ┌─────────────────────┐
│ 升级完成    │ ←── │ 文件拷贝    │ ←── │ 对比Git仓库版本      │
│ 安全移除U盘 │     │ 至U盘目录   │     │ 确定差异文件         │
└─────────────┘     └─────────────┘     └─────────────────────┘
```

## 三、功能模块详细设计

### 3.1 模块一：U盘检测与连接

#### 功能描述
自动检测插入电脑的产品U盘，识别升级目录结构。

#### 目录结构规范
```
产品U盘/
├── IAP/                    # IAP升级文件目录
│   ├── VER.DAT            # 当前版本信息文件（保留）
│   ├── firmware.bin       # 固件文件
│   └── bootloader.bin     # 引导程序
├── RES/                    # 资源文件目录
│   ├── images/            # 图片资源（说明书PNG、图标等）
│   │   ├── icon_1.png
│   │   ├── manual_page1.png
│   │   └── logo.png
│   ├── audio/             # 音频文件
│   │   ├── startup.mp3
│   │   ├── alarm.wav
│   │   └── beep.mp3
│   └── fonts/             # 字体文件
│       └── default.ttf
└── SYSTEM/                 # 系统目录（可选）
    └── config.json
```

#### 关键功能点
| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | U盘自动检测 | 监听USB插入事件，扫描可移动磁盘 |
| 2 | 目录结构验证 | 检查IAP/、RES/目录是否存在 |
| 3 | 版本信息读取 | 读取VER.DAT和VERSION.json |
| 4 | 存储空间检查 | 检查U盘剩余空间是否足够 |

### 3.3 模块三：IAP文件升级

#### 3.1 升级流程
```
┌──────────────┐
│ 开始IAP升级   │
└──────┬───────┘
       ↓
┌──────────────────┐
│ 1. 读取本地版本   │ ←── 读取U盘 IAP/VER.DAT
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 2. 对比云端版本   │ ←── 获取Git仓库 version.json
└────────┬─────────┘
         ↓
┌──────────────────┐    否
│ 3. 需要升级？     │────────→ 跳过IAP升级
└────────┬─────────┘
         │ 是
         ↓
┌──────────────────┐
│ 4. 备份VER.DAT    │ ←── 临时复制到内存或临时目录
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 5. 清理IAP目录    │ ←── 删除除VER.DAT外的所有文件
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 6. 从Git下载新文件│ ←── 克隆或拉取releases/vX.X.X/IAP/
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 7. 恢复VER.DAT    │ ←── 将备份的版本文件写回
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 8. 校验文件完整性 │ ←── SHA256校验
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 9. 升级完成提示   │ ←── 提示用户在设备上手动选择升级
└──────────────────┘
```

#### 3.2 关键操作说明
| 步骤 | 操作 | 详细说明 |
|------|------|----------|
| 备份VER.DAT | 文件复制 | 将U盘IAP/VER.DAT读取到内存或临时目录 |
| 清理目录 | 文件删除 | 遍历IAP目录，删除所有文件，跳过VER.DAT |
| 下载文件 | Git操作 | 使用sparse-checkout或HTTP下载指定目录文件 |
| 恢复VER.DAT | 文件写入 | 将备份内容写回IAP/VER.DAT |
| 校验 | 哈希对比 | 计算下载文件的SHA256，与version.json对比 |

#### 3.3 用户操作说明
> **注意**：IAP文件拷贝完成后，需要提示用户：
> 1. 安全移除U盘
> 2. 将U盘插入设备
> 3. 在设备菜单中选择"系统升级"
> 4. 等待设备完成升级并重启

#### 4.1 资源类型定义
| 类型 | 目录 | 文件格式 | 用途 |
|------|------|----------|------|
| 说明书 | RES/images/manual/ | .png, .jpg | 产品说明书图片 |

| 配置 | RES/config/ | .json, .xml | 资源配置文件 |

#### 4.2 更新流程
```
┌──────────────┐
│ 开始资源更新  │
└──────┬───────┘
       ↓
┌──────────────────┐
│ 1. 获取云端清单   │ ←── 从Git仓库获取version.json中的res_manifest
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 2. 扫描本地资源   │ ←── 遍历RES/目录，计算各文件MD5
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 3. 对比文件差异   │ ←── 比对文件名和MD5值
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 4. 生成更新列表   │ ←── 新增文件 + 修改文件 + 删除文件
└────────┬─────────┘
         ↓
┌──────────────────┐    空
│ 5. 有更新？       │────────→ 跳过资源更新
└────────┬─────────┘
         │ 有
         ↓
┌──────────────────┐
│ 6. 下载差异文件   │ ←── 从Git仓库下载需要更新的文件
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 7. 拷贝到U盘      │ ←── 覆盖或新增到对应RES子目录
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 8. 校验并清理     │ ←── 校验文件，删除过期文件
└────────┬─────────┘
         ↓
┌──────────────────┐
│ 9. 更新完成       │
└──────────────────┘
```

#### 4.3 增量更新策略
```
云端res_manifest:          本地实际文件:
{
    "images/a.png": "m1",      images/a.png (m1) → 一致，跳过
    "images/b.png": "m2",      images/b.png (m3) → 不同，更新
    "audio/c.mp3": "m4",       audio/d.mp3     → 云端无，保留/删除(配置)
    "fonts/e.ttf": "m5"        fonts/e.ttf 不存在 → 新增
}
```

## 四、Git仓库集成方案

### 4.1 仓库配置
| 配置项 | 说明 | 示例 |
|--------|------|------|
| 仓库地址 | Git仓库URL | https://github.com/company/product-updates.git |
| 分支 | 稳定版本分支 | main 或 release |
| 认证方式 | 访问凭证 | Token或SSH Key |
| 本地缓存 | 下载缓存目录 | %AppData%\UpgradeTool\Cache\ |

### 4.2 Git操作方式选型

#### 方案A：Git命令行调用（推荐）
```csharp
// 示例代码
Process.Start(new ProcessStartInfo {
    FileName = "git",
    Arguments = "clone --depth 1 --filter=blob:none --sparse https://github.com/...",
    RedirectOutput = true
});
```
**优点**：简单可靠，无需额外依赖  
**缺点**：需要系统安装Git

#### 方案B：LibGit2Sharp库
```csharp
// 示例代码
using LibGit2Sharp;

var co = new CloneOptions();
Repository.Clone("https://github.com/...", localPath, co);
```
**优点**：纯.NET实现，无需外部Git  
**缺点**：库体积较大

#### 方案C：HTTP API直接下载
```csharp
// 直接下载version.json和指定文件
var http = new HttpClient();
var json = await http.GetStringAsync("https://raw.githubusercontent.com/.../version.json");
```
**优点**：轻量，无需Git  
**缺点**：无法利用Git的版本管理特性

### 4.3 推荐实现
使用**方案A（Git命令行）+ 方案C（HTTP下载）**混合方式：
- 首次运行检查系统是否安装Git，如无则提示安装
- 使用Git获取版本清单和文件列表
- 使用HTTP直接下载大文件，提高效率

## 七、异常处理与风险控制

### 7.1 异常情况处理

| 异常场景 | 处理策略 | 用户提示 |
|----------|----------|----------|
| 网络连接失败 | 重试3次，失败后提示检查网络 | "无法连接到Git仓库，请检查网络" |
| U盘写入失败 | 检查写保护状态，提示更换U盘 | "U盘写入失败，请检查写保护开关" |
| 存储空间不足 | 计算所需空间，提示清理或更换 | "U盘空间不足，需要XX MB" |
| 文件校验失败 | 重新下载，多次失败后报错 | "文件校验失败，正在重新下载..." |
| 升级中断 | 保存状态，支持断点续传 | "升级中断，可重新启动继续" |
| Git认证失败 | 提示检查Token或权限 | "访问Git仓库失败，请检查配置" |

### 7.2 风险防范措施

| 风险 | 应对措施 |
|------|----------|
| **误删文件** | 删除前二次确认，VER.DAT特殊保护 |
| **版本不兼容** | 升级前检查设备型号兼容性 |
| **升级包损坏** | SHA256严格校验，损坏则重新下载 |
| **U盘意外拔出** | 监听USB事件，异常时中止并提示 |
| **并发操作** | 升级过程中锁定操作，防止重复点击 |
