-# H8 升级工具项目

## 一、项目目标

开发一款基于 Windows 的 H8 升级工具。该工具应具备以下能力：

- 自动检测连接到电脑的 H8 U 盘（可移动磁盘，约 256MB）
- 读取并解析 U 盘内本地版本信息
- 从 GitHub 云端获取最新版本信息并与本地版本对比
- 下载最新升级包并解压到临时目录
- 将升级包文件复制到 U 盘目标目录结构中
- 提供清晰的图形界面和状态提示
- 处理常见异常，保证升级过程安全可靠

## 二、核心功能清单

1. 目标U 盘检测（256Mb左右的可移动磁盘 REMOVABLE 类型）
2. U 盘目录结构校验与初始化
3. 本地版本读取与解析
4. 云端版本读取与解析
5. 版本差异对比
6. GitHub 升级包下载与解压
7. 升级文件复制与替换
8. 界面呈现与用户操作
9. 失败重试、网络异常和空间检测

## 三、技术选型

- 开发语言：Python
- UI 框架：PyQt
- Git / 云端访问：GitHub HTTP 下载或 GitHub Releases
- 配置格式：JSON、TXT
- 平台：Windows 7 及以上

## 四、系统架构

整体架构分为以下模块：

- GUI 层：负责界面显示、用户操作和状态提示
- 版本管理层：读取、比较本地/云端版本信息
- GitHub 访问层：获取远程 `version.json`、下载升级包
- 文件与 U 盘管理层：检测 U 盘、校验目录、执行复制操作
- 日志与异常层：记录操作流程，处理错误与重试逻辑

## 五、实施方案

### 5.1 需求分解

#### 5.1.1 U 盘检测与目录校验

- 扫描所有可移动磁盘（REMOVABLE 类型   约 256MB）
- 主要通过大小（256Mb左右）来识别目标 U 盘并读取盘符
- 检查是否存在 `IAP/` 和 `Res/` 目录，若目录缺失，自动创建对应目录。

#### 5.1.2 本地版本读取

- 读取 U 盘中 `IAP/VER.DAT`
- 解析设备型号、软件版本、组件版本、资源版本等信息
- 设备自动将本地版本转换为统一的数据结构
  VER.DAT格式如下（目录位置IAP/VER.DAT）：

```c
P=H8-AP_MAX    //产品名称
T=2026-04-20T10:30:00Z
G=V0.0.0        //总版本号 G_version
1=A3.64.18.B2  //遥控器主板   remote_control
2=B1.64.20.A5  //无线模块(主) RF_host
3=null         //无线模块(从)  RF_slave
4=A2.5A.12.1F  //声呐模块 sonar
5=C0.6C.05.2A  //船控主板  boat_mainboard
6=D5.61.30.0B  //GPS模块  gps
7=E2.64.22.3C  //电调模块  esc
8=F1.63.10.1D  //风扇模块  fan
```

num=NULL 表示为空、该设备没有该模块。

#### 5.1.3 云端版本读取

- 通过解析VER.DAT找到对应GitHub 中的 `产品名-Ve.json`
- 解析 `release_time`发布时间（世界时）、 `G_version`升级包总版本号、`resources` 总升级资源包的大小和下载路径、`IAP_modules`iap各升级模块详细信息等

#### 5.1.4 版本对比与升级决定

- 比较本地软件版本与云端最新版本主要通过比较总版本号（SemVer格式）
- 判断 IAP 文件和 RES 文件是否需要更新
- 生成升级清单，包括需要下载的文件和复制路径

#### 5.1.5 升级包下载与解压

- 从 GitHub 下载最新发布包（例如 `releases/v2.2.0.zip`）
- 下载后解压到临时目录
- 验证文件完整性（可选：MD5/SHA256）

#### 5.1.6 文件复制与更新

- 将解压后更新文件复制到 U 盘对应目录
- 保留 `VER.DAT` 等重要文件的正确格式
- 检查写入是否成功

#### 5.1.7 界面与用户交互

- 主界面显示当前 U 盘信息、当前版本和云端最新版本
- 展示升级项列表和说明
- 显示升级进度、下载状态、日志
- 提供“刷新”“开始升级”“查看详情”“取消”等按钮

#### 5.1.8 异常处理与风险控制

- 网络失败：重试 3 次，失败后提示
- 存储空间不足：预先计算并提示用户
- 文件写入失败：提示并取消升级
- 升级中断：中止操作并回退到稳定状态
- U 盘拔出：检测异常，终止并提示

## 六、详细功能模块说明

### 6.1 U 盘管理模块

输入：系统可移动磁盘列表
输出：目标 U 盘盘符、目录状态

功能点：

- 仅扫描全部可移动磁盘
- 判断磁盘是否为 H8 目标盘（REMOVABLE 类型   约 256MB）
- 查看目标盘内是否有“IAP”“Res”目录，若缺少目录提示用户并创建该目录
- 计算可用空间

### 6.2 本地版本模块

输入：U 盘中的 `VER.txt`
输出：本地版本数据结构

功能点：

- 读取并解析 `VER.txt`
- 提取 `software_version`、`components` 等字段
- 生成升级判断依据

### 6.3 云端版本模块

输入：GitHub 仓库 URL
输出：远程版本配置与最新版本信息

功能点：

- 请求远程 `version.json`
- 解析版本号、文件清单、下载路径
- 支持 GitHub Releases 或仓库 raw 文件

### 6.4 下载与升级模块

输入：升级清单、GitHub 远程资源
输出：本地临时升级包、U 盘目标文件

功能点：

- 下载 ZIP 包或指定文件
- 解压到临时目录
- 校验文件完整性
- 拷贝文件到 `IAP/`、`RES/`
- 更新 `VER.DAT` 或本地描述文件

### 6.5 UI 展示模块

输入：检测结果、升级清单、状态更新
输出：图形界面、进度条、日志文本

功能点：

- 主界面设备信息展示
- 升级内容与说明列表
- 升级过程进度展示
- 完成页面与操作提示

## 八、实施清单（可直接执行）

1. 新建 Python 项目并创建 `src/` 目录
2. 编写 `usb_manager.py`：扫描可移动磁盘、识别 H8 U 盘
3. 编写 `version_manager.py`：解析本地 `VER.DAT`、读取远程 `version.json`
4. 编写 `github_client.py`：下载升级包、处理 GitHub URL
5. 编写 `upgrade_manager.py`：解压、复制和写入升级文件
6. 编写 `gui.py`：构建主界面、进度条、日志输出
7. 编写 `main.py`：启动应用、连接模块
8. 运行并验证“检测 U 盘 -> 比对版本 -> 下载 -> 复制 -> 完成”的完整流程

## 九、风险控制与验收标准

验收标准：

- 能成功检测到插入的 H8 U 盘
- 能读取本地版本并正确显示
- 能获取并解析 GitHub 云端版本信息
- 版本差异比对结果正确
- 可下载升级包并将文件写入 U 盘
- 升级过程中能提示网络/空间/写入异常
- 状态页能正确提示“升级完成”并给出后续操作建议

风险控制：

- 不在 U 盘上直接删除用户数据，优先追加或替换必要文件
- 保留 `VER.DAT` 文件的版本信息与兼容校验
- 严格处理网络异常与用户取消操作
